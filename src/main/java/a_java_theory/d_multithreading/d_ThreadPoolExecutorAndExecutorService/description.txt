Использование ExecutorService и ThreadPoolExecutor позволяет упростить управление потоками и
улучшить производительность за счет повторного использования потоков.

Эти классы и интерфейсы предоставляют мощные средства для управления многопоточностью и асинхронными задачами в Java.



Использование ExecutorService и ThreadPoolExecutor помогает упростить управление многопоточностью в Java,
обеспечивая гибкость и высокую производительность при выполнении задач. Они позволяют легко управлять количеством потоков,
временем их жизни и очередями задач, что делает их незаменимыми инструментами для разработки многопоточных приложений.


Преимущества использования
    * Управление ресурсами: Пул потоков помогает контролировать количество одновременно выполняемых задач.
    * Производительность: Повторное использование потоков снижает затраты на создание новых потоков.
    * Простота: Упрощает код за счет использования высокоуровневых методов управления потоками.


ExecutorService — это интерфейс, который определяет контракт для выполнения задач.
ThreadPoolExecutor — это конкретная реализация интерфейса ExecutorService.

ExecutorService предоставляет основные методы для управления выполнением задач, но детали управления потоками зависят от конкретной реализации.
ThreadPoolExecutor предоставляет подробные настройки для управления пулами потоков, включая количество потоков, время жизни потоков, обработку очередей и стратегию отказа в выполнении задач.
